## h5 Gitar Hero

### x) Lue ja tiivistä.
#### Chacon and Straub 2014: [Pro Git, 2ed: 1.3 Getting Started - What is Git?](https://git-scm.com/book/en/v2/Getting-Started-What-is-Git%3F)

- Toisin kuin jotkin muut versionhallintaysteemit, jotka tallettavat tietoa tiedostoihin tehtyjen muutosten perusteella, git käsittelee dataa tiedostosysteemin snapshottien sarjana
- Commit-komennolla git ottaa snapshotin kaikista tiedostoista ja tallettaa referenssin siihen snapshottiin
- Gitillä voi työskennellä lokaalisti ja tehdä commitit omalle koneelleen, ja vasta myöhemmin lisätä ne etänä olevaan repoon
- Tarkistussummat varmistavat, että tiedostoja ei voida muuttaa ilman, että Git tietää muutoksista
- Tiedostoilla on kolme tilaa:
1. Modified - Tiedostoa on muutettu, mutta sitä ei ole commitettu paikalliseen tietokantaan
2. Staged - Muutettu tieto on merkitty menemään seuraavaan commit-snapshottiin
3. Committed - Data on talletettu paikalliseen tietokantaan
- Workflown kolme tilaa:  
1. Tiedostoja muokataan "working treessä"
2. Valitaan, mitkä muutokset halutaan lisätä seuraavaan commitiin, ne lisätään "staging areaan"
3. Commit vie tiedostot staging areasta ja talletaa ne snapshottina Git-directoryyn (repo)

#### Gitin käyttö on lähinnä 'git add --all && git commit; git pull && git push'. Selitä tuon komennon jokainen osa. Käytä apuna itse valitsemiasi lähteitä ja viittaa niihin.

``git add --all`` - Lisää, muokkaa ja poistaa tiedostoja staging areaan niin, että se vastaa working treen sisältöä.  Lähde: [Git-scm.com - git-add](https://git-scm.com/docs/git-add)  

``&&`` Jälkimmäinen operaatio suoritetaan, jos edellä oleva onnistui. Lähde: [Navigating Command Sequences Like a Pro ](https://dev.to/frnncs/understanding-and-in-linux-bash-navigating-command-sequences-like-a-pro-3apd)  

``git commit`` Lisää muutokset paikalliseen repositoryyn. Lähde: [Git-scm.com - git-commit](https://git-scm.com/docs/git-commit)

``;`` Jälkimmäinen komento suoritetaan, vaikka edellinen komento ei onnistuisi. Lähde: [Navigating Command Sequences Like a Pro ](https://dev.to/frnncs/understanding-and-in-linux-bash-navigating-command-sequences-like-a-pro-3apd) 
 
``git pull`` hakee muutokset repositoriosta ja integroi ne paikalliseen repoon. Lähde: [Git-scm.com - git-pull](https://git-scm.com/docs/git-pull)  

 ``git push`` vie paikalliset muutokset (commitit) toiseen repoon. Lähde: [Git-scm.com - git-push](https://git-scm.com/docs/git-push)

### c) Doh! Tee tyhmä muutos gittiin, älä tee commit:tia. Tuhoa huonot muutokset ‘git reset --hard’. Huomaa, että tässä toiminnossa ei ole peruutusnappia.

Lisäsin paikalliseen repoon uuden md-tiedoston, katsoin git status -komennolla repon tilan ja lisäsin sitten uuden tiedoston staging areaan. Uudella status-komennolla näin, että se odotti nyt committausta. 

![](/images/h5_6dumnotes.png)

En kuitenkaan committanut vaan komennolla ``git reset --hard`` deletoin koko tiedoston, eli se katosi staging areasta ja kokonaan kansiosta. Tämän kanssa kannattaa olla varovainen, ettei tuhoa jotain, mikä olisi pitänyt säilyttää.

![](/images/h5_7notespois.png)


### d) Tukki. Tarkastele ja selitä varastosi lokia. Näytä myös, mitä muutoksia tiedostoihin on tehty. Tarkista, että nimesi ja sähköpostiosoitteesi näkyy haluamallasi tavalla ja korjaa tarvittaessa.

Git log -komennolla voi tarkastella muutoksia. Lähde komennoille: [https://git-scm.com/cheat-sheet](https://git-scm.com/cheat-sheet)

``git log --oneline`` näyttää pelkän commit-viestin.

![](/images/h5_8oneline.png)

Kokeilin siirtää tiedoston uudelle nimelle. Git status -komennolla näkyi, että vanha tiedosto oli deletoitu ja uusi tehty. Sitten kun lisäsin tekemäni muutokset staging areaan, niin nyt Git näytti huomanneen, että kyseessä oli sama tiedosto, joka oli uudellleennimetty.

![](/images/h5_9rename.png)

Muutosten committauksen jälkeen näkyy, että origin/main eli palvelimella oleva repo on jäljessä paikallista repoa.

![](/images/h5_10nocommit.png)

Git push -komennon jälkeen näkyy, että palvelimen repo ja paikallinen repo ovat nyt synkassa.

![](/images/h5_11yescommit.png)
